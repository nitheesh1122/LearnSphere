# Folder Structure & architectural Rules

The *LearnSphere* codebase follows a strict feature-based and architectural separation of concerns.

## `/src/app`
The App Router directory. Contains all routes and layouts.
- **`(auth)`**: Route group for authentication pages (Login, Register). Shared AuthLayout.
- **`(admin)`**: Route group for Administrator dashboard and features. Protected by Admin Middleware.
- **`(instructor)`**: Route group for Instructor dashboard. Protected by Instructor Middleware.
- **`(learner)`**: Route group for the main learning interface. Protected by Learner Middleware.
- **`api`**: Backend API routes (e.g., `/api/auth/[...nextauth]`).

**RULE**: 
- NO business logic in page components. Pages should only fetch data and render Client Components.
- NO direct database calls in Client Components.

## `/src/components`
Reusable UI components.
- **`ui`**: **shadcn/ui** primitives (Button, Card, Input). Do NOT modify logic here, only styles if absolutely necessary.
- **`shared`**: Custom components used across multiple roles (e.g., `CourseCard`, `UserMenu`).

## `/src/lib`
Utility functions and configuration.
- `prisma.ts`: The global Prisma client instance.
- `utils.ts`: Helper functions (shadcn class merger).

**RULE**: All side-effect free logic goes here.

## `/src/hooks`
Custom React hooks (e.g., `useAuth`, `useDebounce`).

## `/src/types`
Global TypeScript type definitions (e.g., API response types, nuanced interfaces not generated by Prisma).

## `/src/styles`
Global styles.
- `globals.css`: Tailwind directives and global variables.

**RULE**: Avoid CSS modules unless absolutely necessary. Use Tailwind utility classes.
